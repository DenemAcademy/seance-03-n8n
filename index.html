C'est quoi n8n ?

En une phrase : c'est un outil qui connecte des applications entre elles et automatise des actions. Tu crées des "workflows" — des chaînes d'actions qui se déclenchent automatiquement.

Imagine : un client t'envoie un email → n8n le détecte → extrait les infos → les ajoute dans ton CRM → t'envoie une notification Slack. Tout ça se fait tout seul, sans que tu touches à quoi que ce soit.

C'est visuel

Pas de code. Tu vois des blocs (les "nodes"), tu les connectes entre eux, tu testes, ça tourne. C'est comme des Legos pour automatiser.

Pourquoi n8n et pas Claude Code ?

Bonne question. Claude Code peut automatiser des trucs aussi. Alors pourquoi un autre outil ?

Parce que les clients veulent voir.

Quand tu livres une automatisation à un client, il veut comprendre ce qui se passe. Avec Claude Code, c'est du code dans un terminal — le client voit rien, comprend rien, fait pas confiance.

Avec n8n, le client voit le workflow. Il voit les blocs, les connexions, le flux. Il comprend : "ah ok, quand un email arrive, ça fait ça, puis ça, puis ça". C'est visuel, c'est clair, c'est rassurant.

Le visuel rassure. Et un client rassuré c'est un client qui paie et qui reste.

Parce que tu vois les logs.

Quand un workflow plante, tu vois exactement où et pourquoi. n8n te montre chaque étape, les données qui passent, l'erreur exacte. Tu debugges en 2 minutes.

Avec un script Claude Code, quand ça plante à 3h du matin... bonne chance pour trouver le problème. n8n te dit "étape 4, erreur 401, clé API expirée". C'est précis.

Les nodes — les briques de base.

Chaque bloc dans n8n s'appelle un "node". Y'en a des centaines. Voilà les catégories.

Le menu des nodes — AI, Actions, Data, Flow, Core, Human in the loop

🤖
AI

Construire des agents autonomes, résumer des documents, chercher dans des bases de données. C'est là que tu connectes Claude, GPT, etc.

🌐
Action in an app

Faire quelque chose dans une app : envoyer un email avec Gmail, poster sur Slack, ajouter une ligne dans Google Sheets, créer un contact HubSpot...

📊
Data transformation

Manipuler, filtrer, convertir des données. Transformer un JSON, merger des listes, extraire des champs.

🔀
Flow

Brancher, merger, boucler. "Si cette condition est vraie → fais ça, sinon → fais ça".

⚙️
Core

Exécuter du code, faire des requêtes HTTP, créer des webhooks. Les briques fondamentales.

👤
Human in the loop

Attendre une validation humaine avant de continuer. "Le client valide → on continue". Pas tout automatiser — garder l'humain quand c'est important.

La configuration d'un AI Agent — prompt, chat model, memory, tools

Comment ça marche

Le node AI Agent a 3 connexions en dessous :

Chat Model *

Le cerveau. Tu connectes Claude (Anthropic), GPT (OpenAI), ou un autre LLM. C'est lui qui réfléchit.

Memory

La mémoire. L'agent se souvient des conversations précédentes. Utile pour les chatbots qui doivent suivre le contexte.

Tool

Les outils. L'agent peut appeler d'autres nodes n8n comme des outils : chercher dans une base de données, envoyer un email, appeler une API. C'est ça qui le rend puissant.

Attention aux credentials

Quand tu exportes un JSON, il contient les noms de tes credentials (pas les mots de passe, mais les noms). Si tu partages le JSON, anonymise les noms avant. Et quand tu importes un JSON d'un repo, tu dois reconfigurer tes propres credentials.

Les bonnes pratiques.

Nomme tes workflows clairement

"Suivi Teliway V13" c'est bien. "My workflow 2" c'est pas bien. Quand t'as 20 workflows, tu dois savoir lequel fait quoi d'un coup d'oeil.

Teste avant d'activer

Clique toujours "Execute workflow" avec des données de test avant de cliquer "Publish". Tu veux pas envoyer 500 emails de relance par erreur.

Utilise les notes

Tu peux ajouter des notes/sticky notes sur le canvas pour expliquer ce que fait chaque partie. Utile quand tu reviens dessus 3 mois plus tard. Ou quand le client veut comprendre.

Un workflow = un process

Fais pas un mega-workflow qui fait 15 trucs. Découpe. Un workflow par process. Plus facile à maintenir, débugger, et expliquer au client.

Sauvegarde tes workflows en JSON

Exporte régulièrement tes workflows en JSON et mets-les sur GitHub. Si un truc plante ou si tu perds ton compte, tu peux tout réimporter.

Les limites.

Le nombre d'exécutions

Sur le Starter à 24€/mois t'as 2 500 exécutions. Si tu as beaucoup de workflows actifs pour plusieurs clients, tu peux atteindre la limite. Surveille dans Settings → Usage and plan.

Les workflows complexes peuvent planter

Plus un workflow a de nodes, plus il y a de chances qu'un truc casse. Découpe en sous-workflows. Un process = un workflow.

Les credentials expirent

Les tokens OAuth (Gmail, Slack) expirent parfois. Tu dois les reconnecter. Surveille tes exécutions — si tu vois des erreurs 401, c'est probablement ça.

La bonne pratique

Teste toujours avec des données réelles avant de publier. Active les notifications d'erreur par email. Et vérifie tes exécutions au moins une fois par semaine.
